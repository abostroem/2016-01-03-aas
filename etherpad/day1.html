three types of verbs: 
    alias: rename a verb variable or make an acronym. a way to make a verb shorter or even a variable or name. way to shorten things
  •     -just type alias to see what you already have aliased

Variables:
    -preceeded by a $
    -can set whatever you want
    -a few specific ones are reserve variables: $PATH; $HOME; $PS1

useful commands/verbs:
    -whoami : identity /name (no noun needed)
    -pwd : location /path (no noun needed)
    -mkdir : make a directory/folder (noun needed: name of directory)
    -ls : list (no noun needed BUT can provide a noun : name of directory you want to list the contents of, even if it isn't the one you're in)
  •     -l : long format (permissions, autors, size, last modified)
  •     -a : show hidden files
    -cd : change directory (noun needed: name of directory)
    -touch filename: create an empty file with the name filename
  •     - updates modification time
    -mv from_location to_location: move file from the from_location (can include path) to the to_location(can include a full path)
    -rm : remove file
    -rmdir : remove directory
    -chmod : change mode
    - Wildcards:
        -for searching or creation
        -? -match ONE unknown character held by the ? position
        - *  - match all characters around * e.g. ls test*.txt returns test.txt, test1.txt, test123.txt
        -{} - match what is inside the brackets e.g. ls test{1, 2}.txt returns test1.txt, test2.txt
  •         - ls test{1,,}.txt is equivalent to ls test1.txt test.txt test.txt
    •   - ranges are specified with .. so from 3 to 6 is expressed {3..6}
  •        - can use ranges to create lots of files at once; not just numbers, letters also (ie, touch test{a..z}.txt will make 26 files)

cat command
  • cat file to see the contents
  • cat > file
    • write stuff in the file
    • control + d to get out (may have to type control+d twice)
    • cat test7.txt > test8.txt puts the contents of test7.txt into text8.txt (test7 is untouched. test8 is overwritten)
    • cat test7.txt >> test8.txt puts the contents of test7.txt into text8.txt (test7 is untouched. test8 is appended onto)
    • head -n# (like 1, 2, 3) filename.txt and tail (same format; give the first or last # lines 
wc: word count
  • get number of lines, number of words, and number of characters in a file
  • wc -w : just get number of words
  • words counts based on spaces
  • lines counts number of new line characters
Pipe:
    | : chains together commands:
        cat test7.txt | sort : run cat test7.txt and pass the output to the sort function. In this case it prints the lines of test7.txt in alphabetical order

PYTHON
immutable versus mutable: 
    immutable definitions don't retroactively change variables 
       a = 3 
       b = a 
       a = 5 # b still equals 3 
    mutable definitions do: 
        a = [1,2,3] 
        b = a 
        a[0] = 5 
        # now both a and b equal [5,2,3] 

brackets matter (), [], {}  
  • a[0] = 5 # indexed a, whereas a(0) means nothing in this case, used typically for functions
  •  

python is 0 index, meaning indices start at 0 
python is object oriented
everything in python is an "object"
example with a python ndarray called data:
    There are attributes like data.size  --> This is some description of something about the object 
    There are methods like data.max()   --> This is a function that performs on things within the object

Nd-arrays:
    You can access parts of an Nd array
    e.g. data[0,0]
    e.g. data[5:10,0:10]  --> This includes the first and excludes the last
    e.g. data[:10,2:] --> No explicit beginning means 0, no explicit end is the last
    e.g. data[5:10,-3:-1]  --> last 3 to last 
    
    Operating on arrays:

    Using methods:
